# Texas Hold'em 서버 — 구현 룰 정리

## 1. 게임 기본 구조

### 방 / 접속
- WebSocket 주소: `ws://localhost:8000/ws/{room_id}/{player_id}`
- 첫 접속 플레이어가 자동으로 방을 생성
- 2명 이상 접속하면 자동 준비(ready) 처리 → 게임 시작
- 핸드 종료 후 3초 뒤 다음 핸드 자동 시작

### 기본 설정 (기본값)
| 항목 | 값 |
|------|----|
| 시작 칩 | 1,000칩 |
| 스몰블라인드 | 10칩 |
| 빅블라인드 | 20칩 |
| 턴 타임아웃 | 30초 |

---

## 2. 페이즈 순서

```
WAITING → PRE_FLOP → FLOP → TURN → RIVER → SHOWDOWN → HAND_END → (PRE_FLOP ...)
```

- **PRE_FLOP**: 홀카드 2장 배분, 블라인드 포스팅 후 베팅
- **FLOP**: 커뮤니티 카드 3장 공개
- **TURN**: 커뮤니티 카드 1장 추가 (총 4장)
- **RIVER**: 커뮤니티 카드 1장 추가 (총 5장)
- **SHOWDOWN**: 남은 플레이어 패 공개, 승자 결정
- **HAND_END**: 폴드로 핸드 종료 (쇼다운 없이 승자 결정)

---

## 3. 베팅 라운드 규칙

### 액션 순서
- **프리플랍**: UTG(빅블라인드 다음 플레이어)부터 시작
- **플랍/턴/리버**: 딜러 왼쪽 첫 번째 살아있는 플레이어부터 시작

### 라운드 종료 조건
- 모든 활성 플레이어가 **자발적으로 1번 이상** 액션했고 (`acted` 집합)
- 모든 활성 플레이어의 **베팅 금액이 동일**할 때

### 빅블라인드 옵션 (BB Option)
- 프리플랍에서 아무도 레이즈하지 않고 콜만 하면
- BB는 반드시 한 번 액션 기회를 받음 (체크 또는 레이즈 가능)

---

## 4. 액션 버튼 규칙

### 오프너 (자발적 베팅 없는 상황)
- 상대방이 아직 베팅하지 않았을 때 (`is_opening = true`)
- 버튼: **CHECK** (해당 시) / **BET** (슬라이더)
- BET → `raise` 액션으로 서버 전송

### 리스폰더 (상대가 베팅한 상황)
- 버튼: **FOLD** / **CALL X칩** / **RAISE** (슬라이더)
- 스택이 콜 금액보다 적으면: **FOLD** / **CALL** 만 표시 (레이즈 없음)

### BET/RAISE 슬라이더 프리셋
| 버튼 | 금액 |
|------|------|
| 1/3 팟 | pot × 0.33 |
| 하프 팟 | pot × 0.5 |
| 3/4 팟 | pot × 0.75 |
| 팟 | pot × 1.0 |
| 올인 | 보유 스택 전체 |

---

## 5. 올인 규칙

### 올인 후 상대방 응답
- 한 명이 올인하면 반드시 상대방이 **콜 / 폴드** 후 라운드 종료
- `is_round_over`는 모든 플레이어가 `acted`에 포함되고 베팅이 같을 때만 `True`

### 올인 런아웃 (All-in Runout)
- 두 플레이어가 모두 올인하면 남은 커뮤니티 카드를 자동으로 깜
- 각 스트릿 사이 **1.5초** 딜레이 (시각적 확인용)
- 추가 액션 없이 플랍 → 턴 → 리버 → 쇼다운 자동 진행

### 스트릿 간 올인 상태 유지
- 새 베팅 라운드(BettingRound) 생성 시 스택이 0인 플레이어는 즉시 `all_in` 복원
- 이렇게 해야 다음 스트릿에도 런아웃이 올바르게 이어짐

---

## 6. 턴 타임아웃

- 각 플레이어의 액션 시간: **30초**
- 클라이언트에 카운트다운 바 표시 (초록 → 빨강 10초 이하)
- 30초 경과 시 자동 **폴드** 처리
- 포스트플랍 스트릿에서도 딜레이 없이 즉시 30초 시작

---

## 7. 패 평가

- 7장(홀카드 2장 + 커뮤니티 5장) 중 **최강 5장** 선택
- 21가지 조합 브루트포스
- 족보 우선순위 (높은 순):

| 순위 | 족보 |
|------|------|
| 1 | 로열 플러시 (Royal Flush) |
| 2 | 스트레이트 플러시 (Straight Flush) |
| 3 | 포카드 (Four of a Kind) |
| 4 | 풀하우스 (Full House) |
| 5 | 플러시 (Flush) |
| 6 | 스트레이트 (Straight) |
| 7 | 트리플 (Three of a Kind) |
| 8 | 투페어 (Two Pair) |
| 9 | 원페어 (One Pair) |
| 10 | 하이카드 (High Card) |

---

## 8. 사이드팟

- 올인 금액이 다른 경우 자동으로 사이드팟 계산
- 각 팟은 해당 팟에 기여한 플레이어만 승리 가능
- 홀수 칩은 딜러 왼쪽 첫 번째 승자에게

---

## 9. 카드 표시 규칙

- 내 홀카드: 항상 앞면 공개
- 상대 홀카드: **항상 뒷면** (쇼다운 전까지)
- 쇼다운 시: 상대 카드 앞면 공개
- 다음 핸드 시작(`deal_hole_cards` 수신) 시 상대 카드 자동으로 뒷면 초기화

---

## 10. 서버 WebSocket 이벤트 요약

### 클라이언트 → 서버
| 이벤트 | 설명 |
|--------|------|
| `join_room` | 입장 (첫 메시지) |
| `ready` | 게임 준비 완료 |
| `player_action` | 액션 (`fold/check/call/raise/allin`) |
| `start_game` | 호스트가 강제 시작 |
| `chat` | 채팅 메시지 |
| `leave_room` | 퇴장 |

### 서버 → 클라이언트
| 이벤트 | 설명 |
|--------|------|
| `player_joined` | 플레이어 입장 알림 |
| `player_ready` | 준비 완료 알림 |
| `deal_hole_cards` | 홀카드 배분 (개인 전송) |
| `game_state` | 테이블 상태 업데이트 |
| `action_required` | 해당 플레이어 액션 요청 |
| `player_acted` | 액션 결과 알림 |
| `showdown` | 쇼다운 결과 |
| `hand_end` | 핸드 종료 (폴드승) |
| `player_left` | 플레이어 퇴장 |
| `error` | 오류 |

## 11. 파일 구조

```
HoldemServer/
├── server/
│   ├── main.py              # FastAPI + WebSocket 라우터, 턴 타이머
│   ├── game/
│   │   ├── card.py          # Card, Deck (Fisher-Yates 셔플)
│   │   ├── hand_evaluator.py# 7장→최강5장, 족보 판정
│   │   ├── betting.py       # BettingRound, 사이드팟, is_round_over
│   │   ├── game_state.py    # FSM, 페이즈 진행, 올인 런아웃
│   │   └── room_manager.py  # 방 생성/참가/퇴장
│   └── tests/
│       ├── test_hand_eval.py# 13개 테스트
│       └── test_betting.py  # 23개 테스트
├── client.html              # 웹 클라이언트 (단일 파일)
├── test_client.py           # 2인 자동 시뮬레이션 스크립트
├── requirements.txt
└── RULES.md                 # 이 파일
```

## 12. 실행 방법

```bash
# 서버 실행
cd server
python -m uvicorn main:app --port 8000

# 웹 클라이언트
# 브라우저에서 client.html 열기 (탭 2개 = 플레이어 2명)

# 자동 시뮬레이션 테스트
python test_client.py

# 단위 테스트
cd server
python -m pytest tests/ -q
```
