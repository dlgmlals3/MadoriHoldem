# 텍사스 홀덤 — 진행 순서 정리

## 1. 기본 역할

| 역할 | 설명 |
|------|------|
| **딜러 (D)** | 버튼을 보유. 포스트플랍에서 마지막으로 행동. |
| **스몰블라인드 (SB)** | 딜러 왼쪽 첫 번째. 강제 베팅 (빅블라인드의 절반). |
| **빅블라인드 (BB)** | 딜러 왼쪽 두 번째. 강제 베팅 (기본값 1,000칩). |
| **UTG** (Under the Gun) | 프리플랍에서 가장 먼저 행동. BB 왼쪽 첫 번째. |

---

## 2. 핸드 진행 순서

```
[블라인드 포스팅] → PRE_FLOP → FLOP → TURN → RIVER → SHOWDOWN
```

### 단계별 흐름

```
① SB 포스팅 (강제)
② BB 포스팅 (강제)
③ 홀카드 배분 (플레이어별 2장)
↓
[PRE_FLOP 베팅]
 - 행동 순서: UTG → UTG+1 → ... → 딜러 → SB → BB
 - BB 옵션: 아무도 레이즈하지 않으면 BB는 체크 또는 레이즈 가능
↓
[FLOP] 커뮤니티 카드 3장 공개
[FLOP 베팅]
 - 행동 순서: SB(또는 딜러 왼쪽 첫 생존자) → ... → 딜러
↓
[TURN] 커뮤니티 카드 1장 추가 (총 4장)
[TURN 베팅] — 순서 동일
↓
[RIVER] 커뮤니티 카드 1장 추가 (총 5장)
[RIVER 베팅] — 순서 동일
↓
[SHOWDOWN] 남은 플레이어 패 공개 → 승자 결정
```

---

## 3. 베팅 라운드 규칙

### 행동 순서
- **프리플랍**: UTG(BB 왼쪽)부터 시작, 시계 방향
- **플랍/턴/리버**: 딜러 왼쪽 첫 생존 플레이어부터 시작, 시계 방향

### 가능한 행동

| 상황 | 선택지 |
|------|--------|
| 아무도 베팅 안 했을 때 | CHECK / BET |
| 상대가 베팅했을 때 | FOLD / CALL / RAISE |
| 콜 금액 ≥ 내 스택 | FOLD / CALL (올인) — RAISE 불가 |

### 베팅 라운드 종료 조건
아래 **두 조건 모두** 충족 시 종료:
1. 모든 활성 플레이어가 **자발적으로 1번 이상 행동**했을 것
2. 모든 활성 플레이어의 **베팅 금액이 동일**할 것

> 레이즈가 발생하면 이전에 콜했던 플레이어도 다시 행동해야 함.

---

## 4. BB 옵션 (BB Option)

프리플랍에서 **아무도 레이즈하지 않고 콜만 한 경우**:
- BB는 반드시 추가 행동 기회를 받음
- BB는 **CHECK** (무료로 진행) 또는 **RAISE** 가능
- 이미 모두 콜한 상태이므로 BB의 체크는 라운드를 종료시킴

---

## 5. 올인 규칙

### 올인 후 상대 응답
- 한 명이 올인 → 상대는 반드시 **CALL** 또는 **FOLD** 후 라운드 종료
- 올인 금액이 이전 베팅보다 작으면 레이즈로 취급하지 않음 (재레이즈 불가)

### 올인 런아웃 (All-in Runout)
- **두 플레이어 모두 올인** → 남은 커뮤니티 카드 자동 공개
- 각 스트릿 사이 1.5초 딜레이
- 추가 베팅 없이 자동 진행: 플랍 → 턴 → 리버 → 쇼다운

---

## 6. 쇼다운 (Showdown)

- 리버 베팅 종료 후 남은 플레이어 전원 패 공개
- 홀카드 2장 + 커뮤니티 5장 중 **최강 5장** 조합으로 승부
- 동점이면 팟을 균등 분배 (홀수 칩은 딜러 왼쪽 첫 승자에게)

### 족보 순위 (높은 순)

| 순위 | 족보 | 예시 |
|------|------|------|
| 1 | 로열 플러시 | A K Q J 10 (같은 무늬) |
| 2 | 스트레이트 플러시 | 같은 무늬 5장 연속 |
| 3 | 포카드 | 같은 숫자 4장 |
| 4 | 풀하우스 | 트리플 + 페어 |
| 5 | 플러시 | 같은 무늬 5장 |
| 6 | 스트레이트 | 5장 연속 |
| 7 | 트리플 | 같은 숫자 3장 |
| 8 | 투페어 | 페어 2쌍 |
| 9 | 원페어 | 페어 1쌍 |
| 10 | 하이카드 | 해당 없음 |

---

## 7. 사이드팟 (Side Pot)

올인 금액이 다를 경우 팟을 분리:
- **메인팟**: 모든 플레이어가 기여한 금액 × 인원
- **사이드팟**: 나머지 금액으로 구성, 기여한 플레이어만 수령 가능

### 예시
A(1,000 올인), B(3,000 올인), C(3,000 콜):
- 메인팟: 1,000 × 3 = 3,000 (A, B, C 모두 수령 가능)
- 사이드팟: 2,000 × 2 = 4,000 (B, C만 수령 가능)

---

## 8. 핸드 즉시 종료 (Hand End)

베팅 라운드 중 **한 명을 제외한 모두가 폴드**하면:
- 쇼다운 없이 남은 플레이어가 팟 전부 수령
- 패를 공개할 의무 없음

---

## 9. 딜러 버튼 이동

- 핸드가 끝나면 딜러 버튼이 왼쪽으로 한 칸 이동
- SB, BB도 자동으로 한 칸씩 이동

---

## 10. 구현상 주의사항 (서버 레벨)

### 액션 순서 인덱스 안정성
- `_active_players()` 는 **폴드된 플레이어를 제외하지 않는다**
- 폴드 추적은 `BettingRound.folded` 에서 담당
- 스트릿 전환 시 `BettingRound` 를 새로 생성하되, `folded` 셋을 **반드시 이전 라운드에서 복사**한다

### 턴 타임아웃
- 각 플레이어 행동 시간: 30초
- 타임아웃 시 자동 폴드 처리 (서버 `turn_timer_loop`)
- 클라이언트도 동일 타이머 표시 (카운트다운 바)
